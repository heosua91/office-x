# GET /guest/availability

### Item
| Item | Description |
| :--- | :--- |
| API Name | Fetch Guest Booking Availability |
| Endpoint | /guest/availability |
| Method | GET |

### Path Parameters
| Field | Type | Required | Note |
| :--- | :--- | :--- | :--- |
| - | - | - | - |

### Query Parameters
| Field | Type | Required | Note |
| :--- | :--- | :--- | :--- |
| invite_token | String | 〇 | The unique token mapped to the smart URL generated by the host. |
| participants_count | Number | － | Used to filter if candidate meeting rooms can hold that many people. |

### Request Parameters (Payload)
| Field | Type | Required | Note |
| :--- | :--- | :--- | :--- |
| - | - | - | - |

### Processing Details
| Phase | Details |
| :--- | :--- |
| 1. Request Validation | - Verify that `invite_token` is present. <br> - Return **400 Bad Request** if missing. |
| 2. Data Acquisition & Verification | - Fetch `meetings` holding the specific `invite_token`. <br> - Return **404 Not Found** if the link does not exist or expired. <br> - Fetch `user_availability` for the targeted host. <br> - Fetch `resource_availability` for the target room (if assigned). |
| 3. State Check | - Perform logical AND checking between the host's free calendar slots and the room's availability for the required meeting duration defined in the token. <br> - Discard rooms dynamically if `participants_count` > `capacity` of `meeting_rooms`. |
| 4. Update Processing | - (No DB updates in this GET request) |
| 5. Response Return | - Return **200 OK** returning a structured array of available dates/time-slots. |

### Response
| Case | Content |
| :--- | :--- |
| Success (200 OK) | ```json<br>{<br>  "success": true,<br>  "data": {<br>    "available_slots": [<br>      { "date": "2026-04-10", "times": ["10:00", "11:00", "15:00"] }<br>    ]<br>  }<br>}<br>``` |
| Error (400) | ```json<br>{<br>  "success": false,<br>  "message": "ERR-CMN-001"<br>}<br>``` |
