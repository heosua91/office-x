## Processing Details

```plain
初期表示 / Initial display
  • Display each item according to the screen item definitions.
  • Set the "Authenticate" (ID: 3) button to a disabled state.

画面更新時 / Interaction
  • When inputting into the "Verification Code" (ID: 2) text box:
    ○ If the input value is exactly 6 half-width digits:
      ▪ Set the "Authenticate" (ID: 3) button to an enabled state.
    ○ Otherwise:
      ▪ Set the "Authenticate" (ID: 3) button to a disabled state.

アクション発生時 / Action
  • **Authenticate button (ID: 3) is pressed**:
    ○ Perform a validation check according to the "Validations" section.
    ○ If there are no validation errors:
      ▪ Execute API request:
        - Endpoint: /auth/register/verify-code
        - Method: POST
        - Request Body: { "email": "...", "code": "..." }
      ▪ If successful:
        - Transition to the next screen (REG-005).
      ▪ If failed (e.g., Code mismatch, expired code):
        - Display the specific error message (Message ID: REG-004-002) below the "Verification Code" text box.

  • **Resend Code link (ID: 4) is pressed**:
    ○ Execute API request:
      - Endpoint: /auth/register/resend-code
      - Method: POST
    ○ If successful:
      ▪ Display a success message (e.g., "Verification code resent").
    ○ If failed:
      ▪ Display an error message (system error).
```

### Queue / Background Tasks
- **Resend Task**: If "Resend Code" is clicked, a background task is triggered similar to REG-002 to send a new OTP via the Email Queue.
