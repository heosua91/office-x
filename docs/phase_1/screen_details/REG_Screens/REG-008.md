## Processing Details

```plain
初期表示 / Initial display
  • Display each item according to the screen item definitions.
  • Set default "Payment Method" (ID: 2) to "Credit Card".
  • Show Credit Card inputs (IDs: 3, 4, 5, 6) and hide Invoice inputs (IDs: 7, 8, 9).
  • Pre-fill Invoice fields (IDs: 7, 8, 9) with data captured in REG-005.

画面更新時 / Interaction
  • When "Payment Method" (ID: 2) is changed:
    ○ If "Credit Card" is selected:
      ▪ Show IDs 3, 4, 5, 6 and hide IDs 7, 8, 9.
    ○ If "Invoice Payment" is selected:
      ▪ Show IDs 7, 8, 9 and hide IDs 3, 4, 5, 6.
  • When inputting values:
    ○ If all visible mandatory fields are filled according to the selected "Payment Method":
      ▪ Set the "Register and Pay" (ID: 11) button to an enabled state.
    ○ Otherwise:
      ▪ Set the "Register and Pay" (ID: 11) button to a disabled state.

アクション発生時 / Action
  • **Back button (ID: 10) is pressed**:
    ○ Transition back to the previous screen (REG-007).

  • **Register and Pay button (ID: 11) is pressed**:
    ○ Perform a validation check according to the "Validations" section.
    ○ If there are no validation errors:
      ▪ Execute API request:
        - Endpoint: /auth/register/payment
        - Method: POST
        - Request Body: Encrypted card token or invoice details.
      ▪ If successful:
        - Transition to the next screen (REG-009).
      ▪ If failed (e.g., Payment rejected, external service error):
        - Display the specific error message returned by the API or external payment service (e.g., "Card declined").
        - Stay on the current screen.
```
