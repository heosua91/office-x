## Processing Details

```plain
初期表示 / Initial display
  • Display each item according to the screen item definitions.
  • Set the "Next" (ID: 8) button to a disabled state.

画面更新時 / Interaction
  • When a plan in "Plan List" (ID: 2) is selected:
    ○ Set the "Next" (ID: 8) button to an enabled state.
  • When inputting into "Promotion Code" (ID: 4):
    ○ If input is not empty:
      ▪ Set the "Apply" (ID: 5) button to an enabled state.
    ○ Otherwise:
      ▪ Set the "Apply" (ID: 5) button to a disabled state.

アクション発生時 / Action
  • **Màn hình vừa load / Immediately after screen load**:
    ○ Execute API request:
      - Endpoint: /auth/register/plans
      - Method: GET
    ○ If successful:
      ▪ Populate "Plan List" (ID: 2) and "Feature Comparison Table" (ID: 3) with current plans and pricing.
    ○ If failed:
      ▪ Display a system error message and potentially provide a "Retry" button.

  • **Apply button (ID: 5) is pressed**:
    ○ Execute API request:
      - Endpoint: /auth/register/check-promo
      - Method: POST
      - Request Body: { "code": "..." }
    ○ If successful:
      ▪ Update "Promotion Application Result" (ID: 6) with discount information and a success message.
    ○ If failed (e.g., Invalid code, expired):
      ▪ Update "Promotion Application Result" (ID: 6) with the specific error message.

  • **Back button (ID: 7) is pressed**:
    ○ Transition back to the previous screen (REG-006).

  • **Next button (ID: 8) is pressed**:
    ○ Perform a validation check according to the "Validations" section.
    ○ If there are no errors:
      ▪ Store the selected plan ID and applied promo code in the application state.
      ▪ Transition to the next screen (REG-008).
```
