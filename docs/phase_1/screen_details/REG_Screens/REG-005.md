## Processing Details

```plain
初期表示 / Initial display
  • Display each item according to the screen item definitions.
  • Pre-populate "Email Address" (ID: 2) with the verified email from the session.
  • Initialize "Number of Participants" (ID: 10) to 1.
  • Display one set of "Participant Name" (ID: 11) and "Participant Email" (ID: 12) inputs.

画面更新時 / Interaction
  • When "Number of Participants" (ID: 10) is changed:
    ○ Re-generate the participant input list (IDs: 11, 12) to match the new quantity (Max 50).
  • When inputting values:
    ○ If all required fields are filled and "Password" (ID: 3) matches "Password Confirmation" (ID: 4):
      ▪ Set the "To Confirmation Screen" (ID: 13) button to an enabled state.
    ○ Otherwise:
      ▪ Set the "To Confirmation Screen" (ID: 13) button to a disabled state.

アクション発生時 / Action
  • To Confirmation Screen button (ID: 13) is pressed:
    ○ Perform a validation check according to the "Validations" section.
    ○ If there are no errors:
      ▪ Temporarily store the input data in the application state.
      ▪ Transition to the next screen (REG-006).
```
