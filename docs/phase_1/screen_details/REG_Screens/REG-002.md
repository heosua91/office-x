## Processing Details

### 初期表示/Initial display
- Display each item according to the screen item definitions.
- Set the "Send Verification Code" (ID: 3) button to a disabled state.

### 画面更新時/Interaction
- When inputting into the "Email Address" (ID: 2) text box:
    - If the input is not empty and matches email format:
        - Set the "Send Verification Code" (ID: 3) button to an enabled state.
    - Otherwise:
        - Set the "Send Verification Code" (ID: 3) button to a disabled state.

### アクション発生時/Action
- **Nút "Send Verification Code" (ID: 3) được nhấn/ "Send Verification Code" button is pressed**:
    - Perform a validation check according to the "Validations" section (e.g., format check).
    - If there are no validation errors:
        - Execute API request:
            - Endpoint: `/auth/register/verify-email`
            - Method: `POST`
            - Request Body: `{ "email": "..." }`
        - **If successful (200 OK)**:
            - Transition to the next screen (REG-003).
        - **If failed (e.g., 400 Bad Request, 409 Conflict)**:
            - Display the specific error message returned by the API (e.g., "Email already registered") or a general system error.



### Queue / Background Tasks
- **Email Queue**: The system pushes a "Send Registration OTP" task to the background worker queue (e.g., Redis/Bull) to handle SMTP delivery asynchronously.
