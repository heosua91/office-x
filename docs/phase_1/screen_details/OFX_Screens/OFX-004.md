## Overview

Screen Name: Client Meeting Details & Edit
URI: /office/meetings/:id
Screen Overview: Edit screen for confirmed client meetings. Allows changes to participants, AI templates, and file attachments up to the meeting time. For Multi-device rooms, supports updating the seating map via drag-and-drop. Allows meeting cancellation resulting in automated notice emails to participants.

## Processing Details

```plain
Initial display
  • Call `GET /office/meetings/:id`.
  • Display read-only info (Client name, guest name, booked slot/format).
  • Populate editable fields (Title, Participants, AI Template, Documents list).
  • If the mapped `meeting_room` is in Multi-device mode, fetch room slots and participants to render the drag-and-drop seat map container.

Interaction
  • Editing fields activates the "Update" button.
  • Dragging a participant on the Seat Map swaps the target seat ID internally in state.
  • Documents section lists files marked as "Uploaded by Guest" (from GRES-005) or "Uploaded by Host".
  • Host retains full authority to view content or permanently delete any document attached to this meeting.

Action
  • "Update" pressed:
    ○ Call `PATCH /office/meetings/:id` with updated title, participants_ids, seat maps, template_id, etc.
  • "Cancel Meeting" pressed:
    ○ Verify intention via red-colored modal.
    ○ On confirm: Call `DELETE /office/meetings/:id` (which handles removing reserve record from local DB & Calendar API, plus dispatching cancellation email templates to the external guest and internal participants).
```
