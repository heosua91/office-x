## Overview

Screen Name: New Internal Booking
URI: /office/meetings/internal/new
Screen Overview: Composite screen connecting personal calendars with meeting room calendars, calculating logical intersections (AND logic) for availability between users and spaces to schedule new internal sessions quickly. Contains selections for AI templates and base file uploads.

## Processing Details

```plain
Initial display
  • Pre-fetch data: user folders, user directory for participants, physical rooms list, and AI templates.
  • Base case selected room is null (online/desk) or pre-allocated.

Interaction
  • When Participants or Room properties changes, front-end fetches availability bounds and projects them over a small linear view "Calendar Cross-view" identifying green (available) time slots.
  • Attached documents validate max sizes on client-side constraints.
  • If a "Multi-device" mode room is targeted by the dropdown, a small notification outlines internal automatic deterministic seating is applied.

Action
  • "Confirm Booking":
    ○ Assembles complex payload properties routing to internal meeting generation API.
    ○ On success routing returns back to OFX-007 indicating the target scheduled meeting inside the respective target folder visually.
```
