## Actions

|---|---|---|---|

## Processing Details

```plain
Initial display
  • Initialize the screen with Current Login ID populated with the user's current email.
  • Disable "Save Button" until "Current Password" is provided.

Interaction
  • Type into the "Current Password" field: "Save Button" becomes active.
  • If "New Password" is typed, "New Password Confirmation" becomes required to match it.

Action
  • "Save Button" button is pressed:
    ○ Call API: `PATCH /auth/profile` with the new data.
    ○ If successful: Show Success Message ID (e.g. SUC-001) and redirect to the settings overview.
    ○ If failed: Show Error Message ID (e.g. ERR-CMN-001 if password incorrect).
  • "Cancel Button" button is pressed:
    ○ Navigate back to the previous view without making changes.
```
