## Processing Details

```plain
Initial display
  • Dispatches `GET /office/settings/availability` to fetch the current setup.
  • Populates the fields and renders the "Temporary Unavail Date/Time List" if unavailable blocks exist.
  • Disables "Save Button".

Interaction
  • When a new unavailable date is added or any checkbox/dropdown is modified, mark form as dirty and enable "Save Button".

Action
  • "Add Temporary Unavail Date/Time" button is pressed:
    ○ Add an empty unsubmitted row for date/time blocking.
  • "Save Button" button is pressed:
    ○ Invokes `POST /office/settings/availability` with the booking rule arrays.
    ○ If successful: Show Success Message ID and navigate back.
    ○ If failed: Show Error Message ID.
```
