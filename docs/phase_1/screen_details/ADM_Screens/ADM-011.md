## Overview

Screen Name: Global AI Limit Policy Setting
URI: /admin/settings/policy
Screen Overview: Configure system-wide AI usage limitation policies. Handles the enabling/disabling of the limiter function, and defines the behavior when a quota is exceeded (auto pay-as-you-go vs. service suspension). Includes a history log of policy changes.

## Processing Details

```plain
Initial display
  • Call API to load current policy settings and history log.
  • If Limiter Function is OFF, the "Action on Overage" options are hidden or disabled.
  • Save Button is disabled until a change is detected.

Interaction
  • Limiter Toggle modified:
    ○ If ON -> Display/Enable "Action on Overage" radio options.
    ○ If OFF -> Hide/Disable "Action on Overage", clearing it if necessary.
  • Any change triggers enabling of the "Save Button".

Action
  • Screen Load:
    ○ Call API to get global settings and history (e.g. GET /tng/settings/policy mapping)
    ○ If successful: Populate fields and history table.
    ○ If failed: Display Error Message ID.
  • Save Button is pressed:
    ○ Ensure Required validations pass.
    ○ Display confirmation modal ("Are you sure you want to enforce these settings?").
    ○ If user confirms modal:
        - Call API: PATCH /tng/settings/policy
        - If successful: Display Success Message, lock UI to new values, reload history.
        - If failed: Display Error Message ID.
  • Cancel Button is pressed:
    ○ Discard unsaved changes and reset UI to loaded DB values.
```
