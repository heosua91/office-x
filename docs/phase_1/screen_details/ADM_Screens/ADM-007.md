## Overview

Screen Name: Detail/Update Basic Plan
URI: /admin/plans/edit
Screen Overview: Can view/change the price and feature permissions for each basic plan. When changing, the application date must be entered. After changing, the new price will be applied in bulk to all currently contracted companies. Allows setting Special Plan flag (hidden status).

## Processing Details

```plain
Initial display
  • Pre-select the tab mapped to the Plan ID passed from ADM-006.
  • Call API to fetch specific plan data and feature definitions.
  • All "New" fields (Fee, Permissions) default to matching the "Current" fields.
  • Before/After Diff Display is hidden.

Interaction
  • Changing Plan Tab:
    ○ Prompts to discard if unsaved changes exist. Otherwise, fetches the new Plan details.
  • Modifying "New Monthly Fee" or "New Permissions":
    ○ Shows and dynamically updates the "Before/After Diff Display" label.
  • "Save button" is enabled only when modifications exist and (if Monthly Fee changed) "Application Start Date" is filled with a valid date.

Action
  • Save button is pressed:
    ○ Validates "Application Start Date" logic (Date >= today).
    ○ If successful: Render Confirmation Modal ("Apply new rate to all current contracts. OK?").
    ○ If failed: Display Error Message ID.
  • Option (Modal) Confirm is pressed:
    ○ Call API: POST /tng/plans
    ○ If successful: Show Success Message (S001), update "Current" view values to new values, clear diff UI.
    ○ If failed: Render Error Message from API.
  • Cancel button is pressed:
    ○ Return to ADM-006. Discard local UI state.
```
