## Overview

URI: /admin/contracts/confirm
Screen Overview: Display the differences (before/after) of changes entered in update/create, and confirm if there are no issues. Navigates back to ADM-001 after execution.

## Processing Details

```plain
Initial display
  • Identify fields with changed values by comparing cached state (from ADM-002) to original record data.
  • Render the Difference Display Table showing Item Name, Before Change (in red), After Change (in green).
  • If New Mode: Render all populated fields with Before Change empty.
  • Notify User in Charge checkbox is empty.

Interaction
  • User can toggle the Notify User in Charge checkbox.

Action
  • Confirm Button is pressed:
    ○ Payload is sent mapping to `PATCH /tng/companies/:id/quota` (or Creation API endpoint).
    ○ Payload includes `notify_user: true/false`.
    ○ If successful: Show Success Message ID (e.g. S001) and transition to ADM-001 (Company Contract List).
    ○ If failed: Display Error Message ID from API response.
  • Modify Button is pressed:
    ○ Return to ADM-002 preventing loss of entered form data.
```
