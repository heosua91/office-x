## Processing Details

```plain
Initial display
  • If New Mode:
    ○ All fields are empty or unselected.
    ○ Set Discount Type to "None" by default.
    ○ Last Updated By / Last Updated At are hidden.
  • If Edit Mode:
    ○ Call API to get specific company details.
    ○ Populate all input fields with existing values.
    ○ Show Last Updated By / At.
  • Verify and populate dropdown options for "Applied Plan" and "Payment Method".

Interaction
  • User toggles "Discount Type":
    ○ If "None", clear "Specific Discount" and disable it.
    ○ If "Fixed amount" or "%", enable "Specific Discount" and mark as mandatory.
  • Evaluate "To Confirmation Screen" button: Enable it when all visually marked required fields hold values.

Action
  • Display mode check:
    ○ If Edit Mode, API called to retrieve existing data. Same for Plan List data.
  • To Confirmation Screen button is pressed:
    ○ Execute frontend validation for all required fields, constraints, max lengths, formats.
    ○ Execute uniqueness check on Email via API if necessary.
    ○ If successful: Cache values into state and transition to ADM-003 Update Confirmation screen.
    ○ If failed: Display field-level Error Messages.
  • Back button is pressed:
    ○ Return to ADM-001 (Contract Company List).
```
