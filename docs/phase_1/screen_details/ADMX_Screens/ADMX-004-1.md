## Processing Details

```plain
初期表示 / Initial display
  • **Immediately after screen load**:
    ○ Execute API request:
      - Endpoint: /admin/billing/status
      - Method: GET
    ○ **If successful**:
      - Display "Current User Status" (ID: 2).
    ○ **If failed**:
      - Show generic loading error.

画面更新時 / Interaction
  • **Input Change (Additional User Count)**:
    ○ Calculate simulation fees locally or via a micro-API.
    ○ Update "Pro-rated Fee" (ID: 4) and "Recurring Fee" (ID: 5) labels.
    ○ Enable "Confirm Request Button" (ID: 8) if count is valid.

アクション発生時 / Action
  • **Confirm Request Button is pressed**:
    ○ Validation: Ensure "Additional User Count" is ≥ 1.
    ○ If validation passes:
      ▪ Execute API request:
        - Endpoint: /admin/billing/user-slots
        - Method: POST
        - Payload: { "count": [Input] }
      ▪ **If successful (200 OK)**:
        - Transition: Navigate to ADMX-004-2.
      ▪ **If failed**:
        - Show error message (e.g., "Payment service unavailable" or "Limit exceeded").
```
